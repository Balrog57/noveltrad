## 2024-06-18 - Missing ARIA labels on dynamic table buttons & secondary dialogs
**Learning:** In PyQt6 applications using custom accessibility wrappers (like `src.gui.a11y.configure`), dynamically generated widgets in table rows (e.g., per-row edit/delete buttons) and secondary dialog inputs often miss accessibility attributes because they aren't caught by static, top-level widget passes. Icon-only buttons like "✏" or "🗑" in tables become invisible or confusing to screen readers without explicit accessible names.
**Action:** Always explicitly call `configure(btn, name=..., tooltip=...)` immediately after instantiating dynamically generated widgets in item views, lists, or tables, ensuring the accessible name includes context (e.g., "Delete project X" instead of just "Delete"). Apply the same to dialog inputs.
