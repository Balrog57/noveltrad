## 2024-05-24 - Accessibility configuration for dynamically generated elements
**Learning:** Secondary dialogs, nested tabs, and dynamically generated elements (such as icon-only/emoji buttons in table rows like `open_btn`, `rename_btn`, and `del_btn`) do not auto-configure for accessibility and require explicit `configure` calls immediately upon instantiation. Furthermore, per-row actions must include contextual information in their accessible names (e.g., "Delete {project_name}") to be meaningful to screen readers when navigating out of context.
**Action:** When adding or modifying interactive elements generated dynamically within loops or table rows in PyQt6, always import and invoke `src.gui.a11y.configure` on each element, injecting the row context into the `name` property to ensure clear accessibility.
